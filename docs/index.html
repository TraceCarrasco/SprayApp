SprayApp Changelog

What's new, fixed, and improved in each update.

v0.4 — Leaderboard & Profiles

Latest March 2026

New Features

  • NewLeaderboard on the profile page with Setters and Sends tabs, showing the top 10 in each category.
  • NewSetters ranked by score = likes × 2 + (likes / climbs) × 10 — likes are weighted above raw volume.
  • NewSends ranked by number of sends; hardest grade shown as a reference column.
  • NewTapping any name in the leaderboard navigates to that user's profile page.
  • NewLiked / ❤ system — heart button left of the Log Ascent button. Likes are counted per setter on the leaderboard.
  • NewSetter name in the climb display is now bold and tappable — opens the setter's profile page.
  • NewUser profile page showing problems set (with count, tappable to full list) and full logbook-style send history.
  • NewSent checkbox when creating a climb — logs an ascent at creation time.
  • NewSort by Hardest and Easiest added to the climb list.

Improvements

  • ImproveRecent activity + leaderboard section is now fully scrollable together.
  • ImproveDefault climb list sort changed to Most Sends.
  • ImproveLeaderboard displays as a clean table (Name | Likes | Climbs / Name | Hardest | Sends) with numbered ranks.
  • ImproveMost Sends sort now uses real ascent array length instead of a stale counter column.
  • ImproveRemoved redundant "Set by:" text below the Log Ascent button.
  • ImproveAscents / comments panel moved higher (60/40 split with the climb image).

Bug Fixes

  • FixDuplicate-key error when double-tapping the bookmark — fixed with an optimistic state update.
  • FixOverflow error on profile page from unbounded ListView — replaced with bounded scrollable layout.
  • FixLikes not counting in leaderboard because liked climbs were filtered by RLS — setter name is now stored directly in liked_climbs at insert time.

v0.3 — Grade Overhaul & Sorting

Early 2026

New Features

  • NewUnified grade display — grade shows setter's grade until ascents are logged, then shows the ceiling average of all grade-feel votes.
  • NewSame grade logic applied to the climb list cards.
  • NewSave / bookmark climbs from the climb detail page.
  • NewDrafts feature — save climbs as drafts before publishing.
  • NewPagination on the climb list for faster load times.

Improvements

  • ImproveAppBar redesigned to show climb name, setter, and grade in a multi-line layout.
  • ImproveBookmark moved to the body next to Log Ascent (Log Ascent centered, bookmark on the right).
  • ImproveDelete button on ascent cards repositioned further right for easier tapping.
  • ImprovePerformance improvements across climb list loading.
