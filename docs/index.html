v0.4 Latest March 2026
Leaderboard, Liked Climbs & User Profiles
New Features
Leaderboard — a Setters tab and a Sends tab, each showing the top 10. Setters are scored by likes weighted against climb count; Sends are ranked by total climbs sent.
Like button — heart icon to the left of Log Ascent. Your likes are counted on the Setters leaderboard.
User profiles — tap any setter's name in a climb or the leaderboard to view their profile: problems set, total sends, hardest send, and full send history.
Sent checkbox when creating a climb — tick "Sent" and an ascent is automatically logged at creation time.
Sort by Hardest / Easiest — two new sort options in the climb list.
Improvements
Recent Activity and the Leaderboard now scroll together as one unified list — no more fixed-height boxes.
Leaderboard displays as a proper table with rank numbers (1–10) instead of medal emojis.
Setter name in the climb detail is now bold and tappable, going straight to their profile.
Default sort on the climb list changed to Most Sends.
Most Sends sort now reads the actual ascent array length instead of a stale database counter.
Removed the redundant "Set by:" label below the Log Ascent button.
Ascents / comments panel moved higher on the climb detail screen (60/40 image split).
Bug Fixes
Double-tapping the bookmark no longer throws a duplicate-key database error.
Profile page no longer overflows when the activity list is long.
Likes now correctly attributed to setters in the leaderboard (setter name stored at like time, bypassing read permission limitations).
v0.3 Early 2026
Grade Overhaul, Bookmarks & Drafts
New Features
Community grading — once ascents are logged, the displayed grade becomes the ceiling average of all grade-feel votes. Before any ascents, the setter's grade shows as default.
Bookmark / Save climbs — save any climb to a personal list accessible from your profile.
Drafts — create a climb and save it as a draft to finish setting up later before publishing.
Improvements
Climb detail AppBar redesigned: climb name is prominent, setter and grade shown beneath in a clean multi-line layout.
Bookmark icon moved from the AppBar into the body — sits to the right of Log Ascent with Log Ascent centered.
Grade logic applied consistently to both the climb detail view and the climb list cards.
Delete button on ascent cards repositioned to the far right for easier one-thumb tapping.
Pagination added to the climb list for faster initial loads.
Performance improvements across climb data fetching.
